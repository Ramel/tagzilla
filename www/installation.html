Installation

Note to Linux users

If you installed Mozilla as root, you must be root to install TagZilla and other such Mozilla extensions to the program directory. In addition, you have to restart Mozilla as root after you've installed TagZilla, so it can update its chrome registry. (That probably goes for other multi-user OSes too, but I don't know which off the top of my head.) If you installed Mozilla into your own home directory, or another directory for which you have write access, this won't be a problem for you.

TagZilla 0.049

Don't let the version number scare you. I've been dogfooding it since I started, so it should be pretty stable. Of course, having said that, you're trying this at your own risk.

TagZilla versions 0.005 and up are dually licensed under the Mozilla Public License and the GNU General Public License.

TagZilla requires JSLib in order to run. Find its installation page here.

Note about the most recent JSLib build: I had to delete my old JSLib install before Mozilla would find the new one. You might want to keep that in mind for your own machine.

Double-check your TagZilla preferences before you try to use it. Your mail settings won't carry over from 0.011 or below.

Uninstallation

No Mozilla program currently provides an easy way to uninstall any extension XPIs you may not like. I recommend that you back up your chrome/ directory and your profile directory before you install any XPI (whether TagZilla or otherwise), so you can "uninstall" it by restoring your backup.
