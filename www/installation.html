Installation

Note to Linux users

If you installed Mozilla as root, you must be root to install TagZilla and other such Mozilla extensions at this time. In addition, you have to restart Mozilla as root after you've installed TagZilla, so it can update its chrome registry. (That probably goes for other multi-user OSes too, but I don't know which off the top of my head.) If you installed Mozilla into your own home directory, or another directory for which you have write access, this won't be a problem for you.

Phoenix Support

TagZilla 0.029 and up has been written to support both Phoenix 0.2 and Mozilla 1.2a. The same XPI should work in both programs. TagZilla's support in Phoenix is limited to textbox support, clipboard support and tagline theft.

Known TagZilla issues in Phoenix:

  • There is no hotkey to launch TagZilla in clipboard mode
  • The menu option to run TagZilla from Phoenix is in the File menu (since there's no Window menu)
  • When you click "Preferences" on the TagZilla window, a scary Mozilla-looking pref window pops up instead of the nice and simple Phoenix one (complete with useless TagZilla mail settings)
These are issues with Phoenix. As it and TagZilla evolve, they're bound to work better together eventually.

Uninstallation

Neither Phoenix nor Mozilla currently provide an easy way to uninstall any extension XPIs you may not like. I recommend that you back up your chrome/ directory before you install any XPI (whether TagZilla or otherwise), so you can "uninstall" it by restoring your backup.
